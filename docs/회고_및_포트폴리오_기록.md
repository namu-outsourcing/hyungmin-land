## 프로젝트 개요

지인(박민식) 소개로 연결된 첫 번째 외주 프로젝트. 대학교 선거 예비후보자(장형민)의 홍보용 랜딩페이지를 제작하는 작업으로, 학생 클라이언트 대상의 소규모 프리랜서 프로젝트다.

| 항목 | 내용 |
|------|------|
| 클라이언트 | 장형민 |
| 프로젝트 성격 | 대학교 선거 후보자 홍보 랜딩페이지 |
| 금액 | 200,000원 (선금 10만 / 잔금 10만) |
| 기술 스택 | HTML/CSS + Vercel 배포 |
| 소통 채널 | 카카오톡 + 노션 |
| 시작일 | 2026-03-11 |

## 오늘 한 것들 (2026-03-11)

### 1. 외주 프로세스 설계
처음 외주를 진행하면서 단계별 흐름을 직접 설계했다.

- 첫 연락 → 요구사항 파악 → 견적 확정 → 선금 → 시안 → 수정 → 납품 → 잔금
- 학생 클라이언트 특성상 선금을 꼭 받아야 한다는 것을 인지 (선거 끝 후 잠수 방지)
- 수정 횟수를 명확히 사전 합의하는 것이 중요함

### 2. 클라이언트 첫 연락 대응
- 지인 소개 관계이므로 딱딱하지 않게 친근한 톤으로 시작
- 첫 메시지에서 바로 질문으로 연결해 대화를 이어가는 방식 채택
- 요구사항은 노션 양식으로 전달 → 한눈에 보기 편하고 클라이언트가 직접 채울 수 있어 효율적이었음

### 3. 요구사항 문서 작성
클라이언트에게 전달할 공식 안내서(Word .docx)를 사전에 준비했다.

포함 내용:
- 프로젝트 개요 및 작업 범위 (포함/미포함 명시)
- 비용 및 결제 구조 (선금/잔금)
- 진행 순서 흐름도
- 요구사항 확인 질문지

안내서는 첫 연락 시가 아닌, 요구사항 파악 후 조건 확정 시점에 전달하는 것이 적절하다고 판단했다.

### 4. 요구사항 파악 완료
노션 양식을 통해 클라이언트 답변을 수령했다.

- 선거일: 6월 3일 (현재 예비후보 기간 → 즉시 사용 필요)
- 페이지 구성: 단일 페이지 스크롤 방식으로 개발자가 추천, 클라이언트 동의
- 디자인: 밝고 따뜻한 / 참신한 / 힘찬 / 붉은 계열 (국민의힘)
- 배포: 카카오톡 링크 + QR코드 (명함에 둘 다 사용)
- 4월~5월 최종 후보 확정 시 소폭 수정 가능성 있음 → 수정 2회 포함 여부 합의 필요

### 5. 도메인 지식 정리 및 클라이언트 안내
도메인 관련 지식이 없는 상태에서 직접 학습하고 클라이언트에게 안내했다.

학습한 내용:
- 도메인과 호스팅의 차이 (도메인 = 주소, 호스팅 = 서버)
- DNS의 개념 (전화번호부 역할 — 도메인과 IP를 연결)
- 서브도메인의 개념과 활용 방식
- Vercel 기본 주소 vs 커스텀 도메인 차이

클라이언트에게 안내한 옵션:
- ① Vercel 기본 주소 (무료, `honggildong.vercel.app`)
- ② 기존 보유 도메인 연결 (추가 비용 없음)
- ③ 새 도메인 구매 (.com 약 3~4만원/년, .kr 약 2만원/년)

## 잘한 점

- 첫 연락부터 단계별 흐름을 설계하고 진행한 것
- 요구사항을 노션 양식으로 구조화해 전달한 것 → 답변이 깔끔하게 정리됨
- 안내서를 사전에 준비해둔 것 (적절한 시점에 전달 예정)
- 도메인 관련 모르는 내용을 즉시 학습하고 클라이언트에게 명확하게 안내한 것
- 영수증, 수정 건 합의 등 사소하지만 나중에 문제가 될 수 있는 부분들을 미리 챙긴 것
